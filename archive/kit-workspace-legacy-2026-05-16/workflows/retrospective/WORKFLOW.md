# Retrospective Workflow

Use at the end of meaningful work to improve the workflow system.

## Process

1. Identify what worked.
2. Identify what failed or was slow.
3. Extract reusable lessons.
4. Decide whether a workflow, runbook, context file, or skill should change.
5. Record only durable improvements; avoid noisy diary entries.

Save in `_workspace/outputs/retrospectives/` when useful.
