# Debugging Workflow

Use when behavior is broken, confusing, flaky, or unexplained.

## Process

1. Capture the observed behavior and expected behavior.
2. Collect exact error output, logs, commands, inputs, or reproduction steps.
3. Form the smallest testable hypothesis.
4. Inspect the nearest relevant code/config first.
5. Change only after evidence points to a cause.
6. Validate the fix against the reproduction.
7. Note root cause and prevention if useful.

Avoid speculative fixes without evidence.
