# Release Notes Workflow

Use to turn commits, diffs, or completed work into user-facing release notes.

## Process

1. Inspect relevant commits or changes.
2. Group by user-visible impact.
3. Separate features, fixes, breaking changes, docs, and internal maintenance.
4. Translate technical commits into clear user-facing language.
5. Mention migration or validation notes when relevant.

Use `_workspace/runbooks/changelog.md` for format guidance.
