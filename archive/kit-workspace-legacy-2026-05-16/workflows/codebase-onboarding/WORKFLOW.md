# Codebase Onboarding Workflow

Use to create or update onboarding docs for a repository.

## Process

1. Inspect top-level structure.
2. Identify apps, packages, services, configs, and tests.
3. Identify setup, dev, validation, and release commands from tooling.
4. Document boundaries and key flows.
5. Update `ARCHITECTURE.md` and `RUNBOOK.md`.
6. Avoid guessing commands; mark unknowns.

## Output

- repository map
- local setup flow
- validation commands
- architecture notes
- risks / missing inputs
