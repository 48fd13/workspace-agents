# QA Test Plan Workflow

Use to design a practical test plan from requirements, specs, or changed behavior.

## Include

- core user flows
- edge cases
- negative cases
- data/state transitions
- permissions and access boundaries
- browser/device/API variants when relevant
- regression risks
- acceptance criteria

## Output

```md
# QA Test Plan

## Scope
## Assumptions
## Test Matrix
## Manual Checks
## Automated Checks
## Risks
```
