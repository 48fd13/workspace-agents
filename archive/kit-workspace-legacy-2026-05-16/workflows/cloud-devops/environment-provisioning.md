# Environment Provisioning Workflow

Use when creating or modifying local/dev/staging/prod-like environments, accounts/projects, namespaces, regions, or service environments.

Use `_workspace/runbooks/environment-provisioning.md`.

Check:

- environment purpose and owner
- naming, tagging, region, and account/project boundaries
- secrets/config sources
- network access and identity permissions
- data isolation and lifecycle
- deployment path and validation
- teardown/cleanup path

Ask before creating or mutating shared or billable resources.
