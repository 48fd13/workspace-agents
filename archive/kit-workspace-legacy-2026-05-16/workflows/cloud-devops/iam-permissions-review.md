# IAM / Permissions Review

Use for IAM roles, policies, service accounts, API tokens, RBAC, groups, permissions, and access-boundary changes.

Use `_workspace/runbooks/iam-permissions-checklist.md`.

Check least privilege, principal scope, resource scope, conditions, privilege escalation paths, secret/token handling, auditability, and rollback.

Treat permission expansion as a stop-and-confirm change.
