# Code / Config Change Workflow

Use for non-trivial local code, config, docs, test, or workflow changes.

## Required context

- `AGENTS.md`
- `_workspace/map/safety-rules.md`
- `_workspace/runbooks/validation.md`
- relevant source/config/docs files

## Process

1. Identify the requested outcome and scope.
2. Inspect only relevant files.
3. State assumptions, risks, and intended files/areas.
4. Ask before proceeding when the work is risky, ambiguous, or gated.
5. Implement the smallest change that satisfies the request.
6. Validate the touched scope.
7. Summarize changes, validation, deviations, and follow-ups.

## Output artifact

For non-trivial work, save a plan or handoff when useful:

- `_workspace/outputs/plans/YYYY-MM-DD-topic-plan-vNN.md`
- `_workspace/outputs/handoffs/YYYY-MM-DD-topic-handoff-vNN.md`
