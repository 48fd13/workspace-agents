# API Contract Change Review

Use when changing public/internal API routes, schemas, events, or SDK-facing contracts.

Use `_workspace/runbooks/api-contract-checklist.md`.

Check compatibility, versioning, auth, validation, idempotency, pagination/filtering, error shape, rate limits, docs/OpenAPI updates, and client migration path.

Output: breaking changes, safe rollout plan, required docs/tests, and risks.
