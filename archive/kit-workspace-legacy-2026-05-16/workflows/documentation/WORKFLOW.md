# Documentation Workflow

Use for generating or improving project docs.

## Required context

- `_workspace/context/documentation-standards.md`
- `_workspace/runbooks/readme-writing.md` when editing README files
- relevant code/config/docs

## Process

1. Identify the audience and action the doc supports.
2. Verify facts from code/config where possible.
3. Put overview in README, operations in runbooks, architecture in architecture docs.
4. Use concrete commands and paths.
5. Mark assumptions explicitly.
6. Keep docs concise and maintainable.
