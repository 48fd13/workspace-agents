# Requirements Clarity Workflow

Use when a request is underspecified, broad, or likely to produce the wrong output without clarification.

## Process

1. Restate the requested outcome in one sentence.
2. Identify missing inputs that materially affect implementation.
3. Separate blockers from assumptions that can be safely made.
4. Ask the fewest high-leverage questions needed.
5. If safe assumptions are enough, state them and continue.

## Output

- Goal
- Known constraints
- Open questions
- Safe assumptions
- Proposed next workflow
