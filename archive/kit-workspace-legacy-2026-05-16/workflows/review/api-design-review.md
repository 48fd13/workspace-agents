# API Design Review Checklist

Use for reviewing API routes, schemas, payloads, and contracts.

Check:

- clear resource naming and HTTP semantics
- request/response shape consistency
- validation and error format
- pagination/filtering/sorting where needed
- authentication and authorization boundaries
- backward compatibility and breaking changes
- idempotency for retries where relevant
- observability and rate-limit considerations

Output concrete findings with path/line and smallest fix.
