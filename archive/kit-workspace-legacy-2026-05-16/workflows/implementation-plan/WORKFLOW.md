# Implementation Plan Workflow

Use before non-trivial code, config, docs, or workflow changes.

## Process

1. Inspect only relevant files.
2. Identify current behavior and target behavior.
3. List intended files or areas to change.
4. Identify risks, validation, and stop gates.
5. Present a concise plan and ask for approval when required.

## Plan template

```md
# Plan

## Goal
## Scope
## Assumptions
## Files / Areas
## Steps
## Validation
## Risks / Stop Gates
```

Save durable plans in `_workspace/outputs/plans/` when useful.
