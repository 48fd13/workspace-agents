# Engineering Principles

- Prefer small, reversible changes.
- Understand existing patterns before changing code.
- Do not add abstractions until the current need justifies them.
- Treat tests, type checks, build checks, and runtime validation as feedback loops.
- If requirements are unclear, clarify before coding.
- If code and docs disagree, trust code and update docs or flag the mismatch.
- Keep security, data integrity, and external contracts explicit in plans.
